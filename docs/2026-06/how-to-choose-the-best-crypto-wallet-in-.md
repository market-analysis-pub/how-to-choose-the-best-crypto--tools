## Security Checklist for Kenyan Users

- [ ] 2FA enabled (SMS or app-based preferred for Kenya)
- [ ] KYC verified (required for M-Pesa on-ramps)
- [ ] Seed phrase backed up offline
- [ ] No funds left in exchange wallets long-term
- [ ] Test small amount before M-Pesa link first use
- [ ] Check wallet regulation status in Kenya (CBK compliance)

## M-Pesa Integration Methods

### Direct Integration
- **Bitpesa**: Native M-Pesa buy/sell
- **Remitano**: Peer offers for M-Pesa payment

### Indirect (P2P Marketplaces)
- **Paxful**: Hundreds of M-Pesa sellers
- **LocalBitcoins**: Community-driven rates
- **Kraken / Binance (via agents)**: OTC options

## Regulatory Notes for Kenya

- Central Bank of Kenya (CBK) crypto stance: cautious, no blanket ban
- M-Pesa wallets must verify identity (KYC tier 2 minimum)
- Self-custody wallets remain unregulated but recommended for security
- Exchange-backed wallets subject to stricter compliance

## Setup Recommendations by User Type

### New to Crypto
1. Start with **Trust Wallet** (intuitive mobile, self-custody)
2. Fund via **Paxful** (easy M-Pesa ramp)
3. Backup seed phrase immediately

### Active Trader
1. Use **MetaMask** for DeFi + bridge swaps
2. Keep trading stack in **Kraken** or **Binance** (liquidity)
3. Move profits to **cold storage** regularly

### Merchant / Remittance
1. Set up **Bitpesa** account (instant M-Pesa settlement)
2. Pair with **Trust Wallet** for diversification
3. Monitor CBK regulatory updates quarterly
